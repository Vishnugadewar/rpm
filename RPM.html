  • Individuals looking to reduce energy consumption

Project Scope

  • Mobile application for electricity usage management

  • Initial focus on residential users

The Energy Management Problem

Visibility Challenges

Most users have no visibility into which devices consume the most energy and typically only become aware of consumption when the monthly bill arrives.

Control Limitations

Difficulty managing energy usage in multi-resident households where manual tracking efforts are perceived as ineffective.

Awareness Gaps

Uncertainty about optimal timing for appliance usage and no clear connection between behavior and energy consumption.

Key Research Questions

  • What are the primary pain points users face when managing home energy consumption?
  • What features would make tracking and reducing energy consumption easier?
  • How do users want AI to assist them in managing electricity usage?
  • How can the application effectively serve households with multiple occupants?

Key Research Findings

Desired Features

  • Real-time device-specific energy tracking
  • Hourly usage and cost tracking
  • Simple, visual dashboards
  • AI-powered usage pattern analysis
  • Automated scheduling based on routines
  • Anomaly detection and alerts

User Preferences

  • Push notifications for excessive consumption
  • Remote device control capabilities
  • Personalized energy-saving tips
  • Clear visual representations of data
  • Proactive but not intrusive AI
  • Manual control options

Market Gaps

  • Most solutions lack personalized AI recommendations
  • Few address multi-user households effectively
  • Limited real-time device-specific data
  • Complex interfaces for non-technical users
  • Poor integration between tracking and actions

Major UX Insights

  • Transparency is Key: Users value visibility into consumption more than automated control.
  • Shared Living Challenges: Multi-resident households face unique energy management issues.
  • Notification Preferences: Users prefer timely, relevant, actionable alerts.
  • Visualization Matters: Simple, clear visual representations are strongly preferred.
  • AI Expectations: Users want proactive but not intrusive AI with manual control.

User-Driven Functional Requirements

Real-time Monitoring

  • Hourly usage measurements
  • Cost tracking by hour
  • Real-time electricity usage
  • Projected monthly bill

"Would like the app to learn my behavior and mimic it" - Mhindi

Smart Notifications

  • Push notifications for limits
  • Excessive consumption warnings
  • Daily/weekly usage alerts
  • Device reminder alerts

"Wants a mobile app with real-time alert features" - Robert

Data Visualization

  • Energy-friendly dashboard
  • Time-based usage tracking
  • Single clear graphs
  • Financial metrics ($)

"Prefers financial metrics with dollar amounts" - Colton

Personalized Features

  • Personalized energy-saving tips
  • Optimal timing suggestions
  • Appliance-specific recommendations
  • Habit tracking

"Simple, clear dashboard with daily usage metrics" - Mouaed

Prototype Evolution

Version 1 Testing

Basic wireframes testing core functionality with focus on navigation and information architecture.

Feedback: "I had difficulty finding the schedule page, and the transition between Auto Mode and Schedule Mode was unclear."

View Version 1 testing

Version 2 Testing

Added interactive elements for more realistic testing of navigation between screens.

Feedback: "I found the UI a bit confusing with two separate options for scheduling."

View Version 2 testing

Version 3 Testing

Refined scheduling functionality and redesigned room/device navigation based on feedback.

Feedback: "The design is generally clear but could add language options for non-English speakers."

View Version 3 testing

Version 4 Testing

Comprehensive prototype with full functionality focusing on AI interactions.

Feedback: "The app should provide more clarity on what Auto Mode and Schedule Mode do."

View Version 4 testing

Final Design Recommendations

High Priority (MVP)

  • Real-time energy monitoring dashboard
  • Device-specific consumption tracking
  • Basic push notifications
  • Simple scheduling functionality

Medium Priority

  • AI-powered recommendations
  • Enhanced visualizations
  • Smart home device integration
  • Improved navigation

Future Enhancements

  • Multi-user profiles
  • Advanced AI capabilities
  • Social sharing features
  • Gamification elements
